- Official doc check (2026-05-28): Supabase Expo quickstart now documents `EXPO_PUBLIC_SUPABASE_URL` and `EXPO_PUBLIC_SUPABASE_PUBLISHABLE_KEY` as the client env vars, while the official React Native quickstart example still shows `EXPO_PUBLIC_SUPABASE_ANON_KEY` for compatibility. Source: https://supabase.com/docs/guides/auth/quickstarts/with-expo-react-native-social-auth and https://raw.githubusercontent.com/supabase/supabase/c8e98cec0e57ce224494552f0908c6af6dde3f4d/examples/auth/quickstarts/react-native/lib/supabase.ts
- Official Supabase Expo/RN bootstrap guidance: create the client with native storage plus `autoRefreshToken: true`, `persistSession: true`, `detectSessionInUrl: false`, and `lock: processLock`; on native, start auth auto-refresh only while the app is active via `AppState`. Source: https://raw.githubusercontent.com/supabase/supabase/c8e98cec0e57ce224494552f0908c6af6dde3f4d/examples/auth/quickstarts/react-native/lib/supabase.ts
- Official Supabase hydration note: the quickstart app reads auth state with `supabase.auth.getClaims()` on launch and again inside `onAuthStateChange`, which is the documented example for session-driven UI hydration. Source: https://raw.githubusercontent.com/supabase/supabase/c8e98cec0e57ce224494552f0908c6af6dde3f4d/examples/auth/quickstarts/react-native/App.tsx
- Official Supabase Expo storage/deep-link constraints: the Expo quickstart recommends Expo SecureStore for mobile (or a custom adapter wrapping it) and warns to keep `expo-secure-store`, `aes-js`, and `react-native-get-random-values` current; email verification and OAuth redirects require native deep linking with an Expo `scheme`, Auth redirect URLs, and `expo-auth-session` / `expo-web-browser` / `expo-linking`. Sources: https://supabase.com/docs/guides/auth/quickstarts/with-expo-react-native-social-auth and https://supabase.com/docs/guides/auth/native-mobile-deep-linking
- Official Supabase session constraint: access tokens are intentionally short-lived (typically 5 minutes to 1 hour), refresh tokens are one-time use, and Supabase recommends not setting JWT expiry below 5 minutes because clients refresh ahead of expiry and shorter windows are vulnerable to clock-skew issues. Source: https://supabase.com/docs/guides/auth/sessions
- Official RevenueCat Expo installation/init guidance: real purchase testing requires an Expo development build; Expo Go only supports RevenueCat preview/mock behavior. Install `react-native-purchases` and `react-native-purchases-ui`, then run a full build instead of relying on hot reload alone. Source: https://www.revenuecat.com/docs/getting-started/installation/expo
- Official RevenueCat SDK init guidance: configure Purchases once early in app lifecycle with the public SDK key only; React Native/Expo must choose separate API keys per platform (iOS vs Android), and RevenueCat explicitly recommends using build configurations or environment variables to select the correct key. Never ship a Test Store key in production. Source: https://www.revenuecat.com/docs/getting-started/configuring-sdk
- Official RevenueCat identity guidance: omitting `appUserID` creates a cached anonymous `$RCAnonymousID:*`; `logIn()` can attach a later known identity; `logOut()` creates a new anonymous identity; custom App User IDs are case-sensitive and project-scoped. For this repo's client mapping, keeping RevenueCat `appUserID` aligned with Supabase `user.id` matches the documented cross-platform identity model. Source: https://www.revenuecat.com/docs/customers/identifying-customers
- Official RevenueCat client-side gating guidance: gate premium UI off `customerInfo.entitlements.active[entitlementId]` / `isActive`; `getCustomerInfo()` is safe to call frequently because the SDK caches state, but entitlement updates are not pushed automatically and should be observed with a customer-info update listener. Source: https://www.revenuecat.com/docs/customers/customer-info
- Official RevenueCat Customer Center guidance relevant to client gating: React Native integration requires `react-native-purchases-ui` 8.7.0 or higher and presents the management UI with `RevenueCatUI.presentCustomerCenter()`. Source: https://www.revenuecat.com/docs/tools/customer-center/customer-center-react-native
